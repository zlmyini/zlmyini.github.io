kali最近用得比较频繁，做一些渗透，但开启很多工具时整个虚拟机就有点绷不住了，原因是内存和硬盘太小（默认内存2G,硬盘20GB）

到虚拟机里面看：

这个sda1快满了，sda1是指硬盘上的第一个分区，通常是一个主分区。主分区是在硬盘上创建的最基本的分区类型之一，可以被格式化为一个文件系统并挂载到Linux文件系统中，用于存储数据和文件。在Linux中，硬盘和分区都被表示为设备文件。例如，sda1设备文件的路径通常是/dev/sda1。

首先在虚拟机设置我们将内存和硬盘都重新设置过：

这里磁盘就是针对接下来虚拟机主分区的扩容准备，这里扩展，自己选择大小（扩展前要先关机并且之前的快照都要删除）

接下来就是进入虚拟机同步物理磁盘分区大小:

打开名sda的硬盘的分区编辑器（parted），对该硬盘进行分区操作步骤如下:

输入 resizepart 1，(如果有其它分区，可以选择其他的序号)，但这里只能选择最右边的分区，因为文件系统是向为未分配的分区扩容，如果比如你有sda2、sda3，那你扩容sda1，它会向右扩容从而覆盖到sda2/sda3分区，是不允许的：

如上图，我这里是因为直接扩容了sda2，给了它40GB。但实际sda2没有挂载，也就并没有实际分配到，所以后面已用和未用都没有数据，所以要扩容一个分区，一定要它已经挂载，下面的步骤是直接扩容sda1,删除了sda2。

卸载完成后如果查看挂载点还有它们，有以下两种情况：

1、被 /etc/fstab文件定义为自动挂载（sda3）这种情况需要编辑/etc/fstab文件，将含有这个错误的分区内容删除或注释：

在编辑器中，找到与sda3相关的行，将其删除或在行的前面添加“#”字符以注释掉该行。然后保存文件并退出编辑器，然后重启即可。

2、可能是未分配的磁盘空间或被误认为是分区（sda2）：

先检查sda2分区的情况：

找到关于sda2的条目，如果它的大小为0或有一些其他的错误，将其删除：

打开fdisk:

输入 'p' 命令来打印磁盘分区信息，确认sda2分区的编号
输入 'd' 命令来删除分区，然后输入要删除的分区编号，例如“2”
最后输入 'w' 命令来保存更改并退出fdisk。

完成上述后，更新磁盘分区表并重新读取分区表：

回到主线，resizepart 1后输入 yes，End 这里填你想扩容大小，因为sda1分区开始是1M左右，结束多大大致就是扩容的大小（这里已经扩容过了显示的是我扩容的大小60G）最后再输入 q (quit) 退出

退出之后，用 df -h 命令看发现没有任何改变，但是使用 lsblk 命令会发现已经扩容成功了，因为这只是 block device 容量变大了，还没有反映到 file system 中

总结一些设计到的命令：

fdisk命令：用于磁盘分区，包括创建、删除、查看磁盘分区信息等。

mkfs命令：用于创建文件系统。

mount命令：用于挂载文件系统。

umount命令：用于卸载已挂载的文件系统。

resize2fs命令：用于扩大或缩小ext2、ext3或ext4文件系统的大小。

fsck命令：用于检查和修复文件系统。

parted命令：用于磁盘分区编辑器，包括对磁盘进行分区、删除、查看分区等操作。

lsblk命令: 列出块设备信息。

这些命令对Kali Linux系统中的磁盘和分区进行创建、编辑、调整和修复等操作。这些命令在日常管理和维护系统时非常有用。
